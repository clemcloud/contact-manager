1. The LIKE Command (Pattern Matching)
Use LIKE when you only know part of a string (text). It uses two special "wildcard" characters:

% (Percent): Represents any number of characters (including zero).

_ (Underscore): Represents exactly one single character.

Examples:
Find names starting with 'J':
WHERE first_name LIKE 'J%' (Matches: James, Jon, Jennifer)

Find emails ending in '@gmail.com':
WHERE email LIKE '%@gmail.com'

Find names that have 'am' anywhere inside:
WHERE first_name LIKE '%am%' (Matches: James, Sam, Pamela)

Find 5-letter names starting with 'J':
WHERE first_name LIKE 'J____' (That's four underscores after the J)

Here is your essential cheat sheet of basic Docker commands for PostgreSQL, formatted so you can easily store or save them for future reference.

### 1. Running and Starting the Container
* **Create and start a new Postgres container:**
  ```bash
  docker run --name my-postgres -e POSTGRES_PASSWORD=clement -p 5432:5432 -d postgres
  ```

* **Start an existing (stopped) container:**
  ```bash
  docker start my-postgres
  ```

### 2. Checking Status and Logs
* **View running containers:**
  ```bash
  docker ps
  ```

* **View all containers (including stopped ones):**
  ```bash
  docker ps -a
  ```

* **Check container logs (useful for troubleshooting errors):**
  ```bash
  docker logs my-postgres
  ```

### 3. Using the Database
* **Access the PostgreSQL interactive terminal (psql):**
  ```bash
  docker exec -it my-postgres psql -U postgres
  ```
  *(You will be prompted to enter the password you set when creating the container)*

### 4. Stopping and Cleaning Up
* **Stop the container:**
  ```bash
  docker stop my-postgres
  ```

* **Remove the container (force delete without stopping first):**
  ```bash
  docker rm -f my-postgres
  ```

* **List downloaded images:**
  ```bash
  docker images
  ```

* **Remove the Postgres image from your local machine:**
  ```bash
  docker rmi postgres
  ```
